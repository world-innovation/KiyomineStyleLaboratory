# AGENTS.md

## Cursor Cloud specific instructions

Real-time private chat app built with Node.js, Express, and Socket.IO. Includes Capacitor for iOS/App Store builds.

### Quick reference

| Task | Command |
|------|---------|
| Install deps | `npm install` |
| Dev server (auto-reload) | `npm run dev` |
| Production start | `npm start` |
| Lint | `npm run lint` |
| Tests | `npm test` |
| Build web for Capacitor | `npm run build:web` |
| Sync to iOS | `npm run cap:sync` |

### Architecture

- **`server.js`** — Express + Socket.IO server with CORS support for native apps. Rooms are in-memory (no database). Serves static files from `public/`.
- **`public/`** — Frontend: lobby (`index.html`) and chat room (`chat.html`) with vanilla JS. PWA manifest included.
- **`capacitor.config.ts`** — Capacitor config for iOS builds. Set `server.url` to the deployed server URL before building.
- Rooms are created via `POST /api/rooms` and accessed by 6-char hex invite codes.
- Empty rooms are auto-cleaned after 30 minutes of inactivity.
- Frontend JS uses `window.CHAT_SERVER_URL` to support both web (same-origin) and native (remote server) modes.

### Caveats

- The dev server (`npm run dev`) uses `node --watch` (Node 22+). Hot-reload watches all `.js` files; CSS/HTML changes are instant on browser refresh.
- Tests in `test/` spawn a child process on port 3000. Make sure port 3000 is free before running `npm test`.
- There is no persistent storage — all rooms and messages are lost on server restart.
- iOS builds require macOS + Xcode. See `README.md` for the full App Store release guide.
- `www/` and `ios/` are gitignored — they are generated by `npm run build:web` and `npx cap add ios`.
